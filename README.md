# 🗝️ 功能简介

## ⚡ 一键生成副本大纲

点击生成后，输出完整副本结构：

* 基础信息（名称 / 评级 / 标签）
* 队伍结构（含内鬼机制）
* 世界背景（表层 → 真相层）
* 生存机制（主机制 / 隐藏规则）
* 场景 / 道具 / NPC / Boss
* 反转结构（3~5 个张力节点）

👉 **重点：这是“副本系统”，不是故事文本**

---

## 🎭 用户特殊身份（V2）

{{user}} 可以是：

* 玩家
* NPC
* Boss
* 系统

👉 勾选后会**强制覆写所有逻辑**

---

## 🔮 世界观融合 / 聊天注入

可自动读取：

* 世界书内容
* 角色设定
* 最新聊天记录

👉 用于“剧情进行中 → 动态生成副本”

---

## 📖 自动注入世界书

生成后可直接写入 Worldbook：

* 自动包裹结构
* 自动控制深度
* 用于后续剧情驱动

---

## 📦 预设系统（V2）

支持：

* 导出 JSON
* 导入他人配置
* 多维预设（生成器 / prompt / 注入）

# 🧠 生成器与提示词解析

---

## 一、生成器参数影响逻辑

### 1. 评级（最重要）

👉 对应 MODULE 2：

* 决定：

  * 难度
  * 真相层数
  * 死亡率
  * Boss阶段

---

### 2. 人数

影响：

* 信息分配
* 内鬼机制
* 场景复杂度

---

### 3. 模式

* PVE：合作
* PVP：对抗
* 混合：前合作后背刺

---

## 二、Prompt 结构解析（核心）

你的 Prompt 分为：

---

### MODULE 0：输入层

👉 来自生成器配置

---

### MODULE 1：知识库

👉 无限流“规则体系”

可自行扩展（非常推荐）

---

### MODULE 2：约束层（最关键）

👉 和生成器完全绑定：

* 评级约束
* 人数约束
* 模式约束

👉 本质：**控制生成质量**

---

### MODULE 3：输出模板

👉 定义结构：

必须包含：

* 世界
* 机制
* NPC
* Boss

---

### MODULE 4：输出格式

⚠️ 必须保留：

```id="mustformat"
<dungeon>...</dungeon>
<dungeon_info>...</dungeon_info>
```

否则：

👉 脚本无法解析

---

## 三、Prompt 修改指南

可以改：

* MODULE 1（知识库）
* MODULE 3（输出细节）

不建议改：

* MODULE 2（会破结构）
* 标签结构

# ❓ 常见问题

---

### Q1：为什么生成质量不稳定？

👉 原因：

* API 模型不稳定
* Prompt 被改坏
* 世界书冲突

---

### Q2：为什么没有生成完整结构？

👉 检查：

* 是否包含 `<dungeon>`
* token 是否不足

---

### Q3：为什么世界书没生效？

👉 可能：

* 没挂载
* 深度冲突

---

### Q4：API 报错怎么办？

👉 检查：

* URL 是否带 `/v1`
* Key 是否正确
